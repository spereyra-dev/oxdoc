# Support

Use GitHub issues for bug reports, feature requests, and design discussions.

Before opening an issue:

- Check whether the behavior is within scope. `oxdoc` extracts data; it does not render Office documents.
- Include the file type, command, expected output, and actual output.
- Share a minimal sample document when it is safe to redistribute.
- Avoid attaching private or sensitive documents.

Security issues should follow `SECURITY.md` and should not be opened as public issues.
